An overview of how Orchestra register works

The basic idea is that you have some of players, and some events like rehearsals and a concert, and you want to know which players are planning to be at which events.

Players

As far as this system in concerned, payers have an first name, last name, email address, and the part (instrument) they play. parts are further divided into sections like strings or woodwind. The list of available parts and sections, and the order in which they should be displayed, can be set when the system is installed.

The system also stores the information required to let players log in to the system, and what level of access each player has.

Although the system stores each players' email address and full name, this information is only visible to people with speciall access rights. In the publicly visible pages, it only shows a player's firstname and last initial, along with which part they play.

Events

An event has a name, description, venue, start and end time (and date).

The list of events can be mananged by users with an appropriate access level. There are facilities for exporting the list of events so they can be imported into diary software, and into a MediaWiki page.

The register

The register lists events across the top, and players down the side. In each cell of the register, it shows the players' attendance as one of:

  • -: no information has been input yet.
  • No: the player cannot attend
  • Unsure: the player is unsure whether they will be able to attend
  • Yes: the player intends to attend
  • Not needed: the player is not needed for this rehearsal

The attendance information is shown nicely colour-coded.

How users get access to the system

Anyone who know the right URL can go to the register and see the attendance information. This is a read-only view that should print nicely.

Players can get access to edit their own information using a special URL with a token on the end, like ...?t=VDcHOaWWq0XHUwXDD6wJZwzKTqbciiKwoFrToTZ8. Each player will have a different token, and if they bookmark that URL, then they can get access to edit their own attendance information.

People with a higher level of access than that have to log in with an email address and password. There are two classes of people with a higher level of access: Organisers can edit any player's attendance information, and edit the list of events and players. Adminsitrators can do anything in the system.
